01 · The mental model

先合并事件，再比较热度

同一新闻可能被多个账号原创、引用、回复或转发。系统先把它们归入一个 Topic，避免同一件事重复占榜。

01采集 Post原创、引用、回复、转发分别保存
02归入 Topic同一事件或洞察合并到一起
03刷新指标每 4 小时抓取最新互动快照
04折算规模主 Post 全算，其他 Post 打 3 折
05计算速度与上次快照比较每小时增量
06生成排名规模和速度各占 50%
◎
一句话理解：大事件有规模分，突然爆发的新事件有速度分；两者兼具，才会靠近榜首。

02 · The formulas

两步得到原始指标，再做相对排名

热度是 0–1 之间的相对值。0.90 不是“90 分内容”，而是它在当前候选 Topic 中处于很靠前的位置。

1

有效浏览量：减少重复受众

浏览量最高的 Post 全额计入；同 Topic 的其他 Post 只计 30%，保留扩散信号但不过度累加。

有效浏览量 = 最高单帖浏览量
＋ 其他 Post 浏览量之和 × 30%
2

增长速度：识别正在爆发

比较相邻两次有效快照。若接口修正导致浏览量下降，本轮增量按 0 处理，不产生负速度。

速度 = max(本次有效浏览量 − 上次有效浏览量, 0)
÷ 间隔小时数

最后：转换为百分位并 50 / 50 合成

系统分别对 log(1 + 有效浏览量) 和 log(1 + 增长速度) 排序并计算 0–1 百分位。这样最终比较的是相对位置，而不是让绝对数量直接按倍数碾压。

浏览规模 50%增长速度 50%
heat = viewScore × 0.5 ＋ velocityScore × 0.5

05 · Lifecycle

热榜不是永久榜单

每轮指标刷新都会重新计算。新 Topic 可以上榜，降温 Topic 会经历确认后下榜，过期 Topic 会停止追踪。

上榜门槛有效浏览量达到 1,000,000。热度决定达到门槛后的相对顺序。
下榜缓冲连续两次 heat < 0.35 才下榜，避免单次接口波动导致频繁进出。
停止追踪连续两次增长率 < 5%，或 Topic 中所有 Post 均超过发布后 72 小时。

06 · Know the boundary

它计算什么，也要知道它不计算什么

这是一个确定性的传播热度算法，不由 LLM 决定分数。

当前已使用

  • Topic 下各 Post 的浏览量
  • 相邻快照之间的浏览增量与速度
  • 候选 Topic 之间的相对百分位
  • 连续低热度、低增长和 72 小时状态规则

当前未进入热度公式

  • 点赞、回复、转发和引用——会保存，但暂不加权
  • 发布者声望或账号粉丝数
  • LLM 主观判断、人工置顶或地域偏好
  • 异常流量识别和个性化推荐
